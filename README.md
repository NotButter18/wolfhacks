# Synapse

A hospital load prediction engine built for the city of Brampton, designed to align with Brampton's 2040 Smart City Vision. Built at WolfHacks.

## What it does

Synapse visualizes and predicts patient load on Brampton hospitals in real time. You can adjust the current load level — low, medium, or high — as well as simulate a Condition X scenario (mass casualty or surge event) to see how the system responds under pressure.

Based on the current inputs, Synapse analyzes what is happening across the hospital network and generates recommended solutions to manage the load. Those recommendations can then be fed directly back into the prediction engine to model the outcome, showing you the projected impact before any real decisions are made.

The goal is to give hospital administrators and city planners a data-driven tool to get ahead of capacity issues rather than react to them.

## Features

- Adjustable load levels: low, medium, and Condition X
- Live visualization of hospital patient flow and capacity
- Automatic solution recommendations based on current conditions
- Feed recommended solutions back into the engine to preview results
- Data dashboard showing what is currently happening across the system

## Tech stack

- Frontend: HTML, CSS, JavaScript
- Backend / data processing: Python

## Background

This project was built in response to Brampton's ongoing hospital capacity challenges. Brampton is one of the fastest growing cities in Canada and currently has one of the lowest hospital bed-to-population ratios in the country. Synapse was designed with Brampton's 2040 vision in mind — using data and smart infrastructure to build a more resilient city.
